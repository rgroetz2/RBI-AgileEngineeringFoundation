# Definition of Ready
A backlog item is Ready when:
- Acceptance criteria are defined and testable.
- Dependencies are clarified.


A backlog item is considered ready, when ALL of the following criteria are met:
User Story Quality
- Story follows the format: "As a [role], I want [feature], so that [benefit]"
- Story is written from the user's perspective (not technical implementation)
- Story is small enough to complete within one sprint
Acceptance Criteria
- At least 3-5 specific, testable acceptance criteria are defined
- Acceptance criteria use verifiable language ("User can...", "System displays...", "Data is stored...")
- Criteria cover both happy path and edge cases
Team Alignment
- Story has been discussed in backlog refinement session
- Development team has no blocking questions
- Scrum Master confirms story meets DoR standards
- PO is available to answer questions during sprint


# Definition of Done
A backlog item is Done when:
- All acceptance criteria are met.
- Code is in local repository.
- Feature is running locally to production.
- PO has verified the feature on local dev WS.
- Documentation and test evidence are updated.
