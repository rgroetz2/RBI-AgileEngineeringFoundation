# ISTQB Foundation Level (CTFL) – Syllabus Summary

## 1. Fundamentals of Testing

### 1.1 Why is Testing Necessary?
- Detect defects and reduce risk
- Ensure requirements are met
- Build confidence in system quality
- Prevent defects early (shift-left approach)

### 1.2 What is Testing?
- Testing = verification + validation
- Includes static (reviews) and dynamic (execution) activities
- Objective: find defects, not prove absence

### 1.3 Testing Principles
1. Testing shows presence of defects
2. Exhaustive testing is impossible
3. Early testing saves time and cost
4. Defects cluster together
5. Beware of pesticide paradox
6. Testing is context dependent
7. Absence-of-errors fallacy

### 1.4 Test Process
- Test Planning
- Test Monitoring & Control
- Test Analysis
- Test Design
- Test Implementation
- Test Execution
- Test Completion

### 1.5 Psychology of Testing
- Tester mindset: critical, curious, skeptical
- Avoid conflict with developers
- Collaboration improves results

## 2. Testing Throughout the Software Lifecycle

### 2.1 Software Development Models
- Waterfall
- V-Model (test levels aligned with dev phases)
- Agile (iterative, incremental, continuous testing)

### 2.2 Test Levels
- Unit Testing
- Integration Testing
- System Testing
- Acceptance Testing

### 2.3 Test Types
- Functional Testing
- Non-functional Testing (performance, usability, security)
- Structural Testing
- Change-related Testing (retesting, regression)

### 2.4 Maintenance Testing
- Triggered by changes, fixes, or environment updates
- Includes regression testing

## 3. Static Testing

### 3.1 Basics of Static Testing
- No code execution
- Detect defects early
- Includes reviews and static analysis

### 3.2 Review Process
- Planning
- Kick-off
- Individual review
- Review meeting
- Rework
- Follow-up

### 3.3 Review Types
- Informal Review
- Walkthrough
- Technical Review
- Inspection

### 3.4 Success Factors
- Clear objectives
- Trained participants
- Management support

### 4.1 Test Techniques Overview

- White-box (structure-based)

- Boundary Value Analysis
- Decision Table Testing
- State Transition Testing

### 4.3 White-box Techniques

### 4.4 Experience-based Techniques

- Exploratory Testing

### 4.5 Collaboration-based Approaches

## 5. Test Management

### 5.1 Test Planning
- Define scope, objectives, risks
- Estimate effort
- Assign resources

### 5.2 Risk Management
- Product risks (quality issues)
- Project risks (schedule, resources)
- Risk-based testing prioritizes high-risk areas

### 5.3 Test Monitoring & Control
- Track progress using metrics
- Adjust plan as needed

### 5.4 Configuration Management
- Version control of testware
- Traceability between requirements and tests

### 5.5 Defect Management
- Defect lifecycle: New → Assigned → Fixed → Retested → Closed
- Report clearly with reproducible steps
