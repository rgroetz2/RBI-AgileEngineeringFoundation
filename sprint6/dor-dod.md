# Definition of Ready
A backlog item is Ready when:
Acceptance criteria are defined and testable.
Dependencies are clarified.
The item is estimated and prioritized.
Design or UX assets (if needed) are available.

A backlog item is considered ready, when ALL of the following criteria are met:
User Story Quality
[ ] Story follows the format: "As a [role], I want [feature], so that [benefit]"
[ ] Story is written from the user's perspective (not technical implementation)
[ ] Story is small enough to complete within one sprint
[ ] Story has a clear business value articulated
Acceptance Criteria
[ ] At least 3-5 specific, testable acceptance criteria are defined
[ ] Acceptance criteria use verifiable language ("User can...", "System displays...", "Data is stored...")
[ ] Criteria cover both happy path and edge cases
[ ] Volodymyr has tested acceptance criteria for testability
Dependencies & Risks
[ ] All external dependencies identified (APIs, third-party services, data sources)
[ ] Technical dependencies clarified with development team
[ ] Blocking issues resolved or clearly documented
[ ] Known risks documented with mitigation strategies
Estimation & Prioritization
[ ] Story has been estimated by the development team (using T-shirt sizes or story points)
[ ] Story is prioritized in the backlog by PO
[ ] Team agrees the story is achievable within the sprint
Design & Assets
[ ] UI mockups or wireframes available (for frontend stories)
[ ] Design assets follow accessibility guidelines (for F5 stories)
[ ] API contracts defined (for integration stories)
[ ] Copy/content approved (for text-heavy features)
Team Alignment
[ ] Story has been discussed in backlog refinement session
[ ] Development team has no blocking questions
[ ] Scrum Master confirms story meets DoR standards
[ ] PO is available to answer questions during sprint


# Definition of Done
A backlog item is Done when:
All acceptance criteria are met.
Code is merged into the official repository.
Feature is deployed to production.
PO has verified the feature live on production.
Documentation and test evidence are updated.
