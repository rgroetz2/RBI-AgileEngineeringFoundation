# Hnads-On Task: Feature Exploration and T1&T5 Test Design

## Objective

Explore the system under test to identify available features and
functions and use the findings as the basis for structured T1&T5
test design.

---

# Task Description

## Part 1 - System Exploration

Explore the system under test using exploratory testing techniques.

Your goal is to identify and document:
- Features
- Functions
- User workflows
- Business capabilities

Create a structured overview of the discovered functionality.

### Examples
- Login
- Registration
- Search
- Shopping cart
- Checkout
- User profile management
- Product filtering

---

## Part 2 - Select a Use Case

Choose one realistic use-case that uses one or more identified
features/functions.

### Example Use-Cases
- Customer purchases a product
- User resets password
- Customer searches and filters products
- User updates profile data

---

## Part 3 - Create T1&T5 Test Set

Based on the selected use-case, create a complete T1&T5 test set.
The title of the test should be according to the title-syntax.

(e.g. T1_LOGIN_withValidCredentials_dashboardIsDisplayed)

---

# Expected Outcome

## Deliverables
- List of discovered features/functions
- Selected use-case description
- Structured T1&T5 test set
