# Hands-On: T1&T5 Test Design for Acceptance and Regression Testing

## Objective

Apply the T1&T5 test design technique to create a structured test set
for acceptance and regression testing and execute the tests. 


Your task is to analyze the impact of these changes from Sprint 6 and create a
structured T1&T5 test set that supports:
- Acceptance testing of the new functionality
- Regression testing of affected existing functionality

## Task Description

### Part 1 - Analyze the Change

Review the provided user stories and identify:
- Newly implemented functionality
- Changed functionality
- Potentially impacted existing features
- Critical business workflows
- Integration points
- Risk areas

The implemented user stories you can find here:
https://github.com/rgroetz2/RBI-AgileEngineeringFoundation/tree/main/tbll-learningJourneys/lj-005-2026-HolisticTestingWithGenAI-GOIT-BAbootcamp/sprint6/sprint6-userStories

The sprint goal here:
https://github.com/rgroetz2/RBI-AgileEngineeringFoundation/blob/main/tbll-learningJourneys/lj-005-2026-HolisticTestingWithGenAI-GOIT-BAbootcamp/sprint6/sprint6-sprintGoal.md

### Part 2 - Define the Test Scope

Determine:
- Which functionality requires acceptance testing
- Which existing functionality requires regression testing
- Which business processes are most critical
- Which risks should be prioritized

### Part 3 - Create T1&T5 Test Set

Create a structured T1&T5 test set for one use-case for:
- Acceptance testing
- Regression testing

### Part 4 - Execute the tests
- Execute your tests and document your findings on post-its.
