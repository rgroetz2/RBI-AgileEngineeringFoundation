# Prompt: Exploratory Feature Discovery and T1&T5 Test Design

You are acting as a quality engineer and exploratory tester.

The system under test is the following web shop for tools and hardware:

- Webshop: https://practicesoftwaretesting.com/
- GitHub Repository: https://github.com/testsmith-io/practice-software-testing
- Official Feature List:
  https://github.com/testsmith-io/practice-software-testing/blob/main/docs/features.md

The T1&T5 test design approach is described here:
https://github.com/rgroetz2/RBI-AgileEngineeringFoundation/blob/main/tbll-learningJourneys/lj-005-2026-HolisticTestingWithGenAI-GOIT-BAbootcamp/sprint6/sprint6-execution/T1T5-testdesign-description.md

## Part 1 - Exploratory Feature Discovery

Explore the webshop and analyze the provided feature list.

Create a structured overview of:
- Features
- Functions
- User workflows
- Business capabilities
- Important user interactions
- Critical business processes

Group the discovered functionality into logical categories.

Examples:
- Authentication
- Product Search
- Shopping Cart
- Checkout
- User Profile
- Administration
- Favorites/Wishlist
- Payments

## Part 2 - Select a Use Case

Choose one realistic business-critical use-case that uses multiple features/functions.

## Part 3 - Create T1&T5 Test Titles

For the selected use-case, create a structured T1&T5 test set (one T1, one T2, one T3, one T4, one T5) in mark down format.

Important:
- Create ONLY test titles according to the T1T5 title syntax
- Do NOT create detailed test steps
- Do NOT create expected results
- Do NOT create preconditions
- Do NOT create test data details
