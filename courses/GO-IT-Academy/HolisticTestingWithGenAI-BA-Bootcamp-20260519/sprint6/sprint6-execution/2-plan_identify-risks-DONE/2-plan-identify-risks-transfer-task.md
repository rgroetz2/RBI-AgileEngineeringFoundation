
# Transfer Task: Risk-Based Test Planning with GenAI

## 1. Description

Apply risk-based test planning using GenAI to evaluate and prioritize
upcoming user stories for testing.

Participants learn how to:
- Assess testing risks
- Prioritize testing activities
- Use AI-supported risk analysis
- Compare outcomes from different GenAI models
- Apply the MoSCoW prioritization method

Your task is to use `(RBI)(Chat)_GPT` and different GenAI models to:
- Analyze the user stories
- Assess their risks
- Prioritize testing activities
- Compare AI-generated prioritization outcomes

---

## 2. Background

You are part of a product team preparing the next sprint.

Due to limited testing capacity, not all stories can receive the same testing
focus.

## Task Description

### Part 1 - Analyze the User Stories

Review the provided user stories (see sprint6-content.md) and the sprint goal (see sprint6-content.md).

Consider:
- Business impact
- Technical complexity
- Integration risks
- Security relevance
- Production impact
- User visibility
- Regression potential

---

### Part 2 - Prioritize Using MoSCoW

Prioritize the stories using:

- MUST (2)
  - Critical
  - Must be tested before release

- SHOULD (1)
  - Important
  - Test if time allows

- COULD (2)
  - Nice to have
  - Optional coverage

- WON’T (3)
  - Out of scope
  - Not worth testing this sprint

Document your reasoning.

---

### Part 3 - Compare Different GenAI Models

Use different GenAI models and compare:
- Risk assessment results
- Prioritization decisions
- Reasoning quality
- Identified risks
- Suggested testing focus
