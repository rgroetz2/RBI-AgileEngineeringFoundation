# Apply Fundamentals of Testing on Toolshop

## 1. Title
Apply Fundamentals of Testing on Toolshop

## 2. Syllabus Reference
ISTQB FL – 1 Fundamentals of Testing

## 3. Learning Objective
Apply the fundamentals of testing to one Toolshop user journey and produce a practical artifact that supports sprint quality decisions.

## 4. Context / Scenario
Your Scrum team is preparing the next Toolshop increment at https://practicesoftwaretesting.com/. You need to apply core testing fundamentals early to reduce avoidable defects and improve release confidence.

## 5. Task Instructions
1. Open https://practicesoftwaretesting.com/ and select one end-to-end flow (for example: browse products, add to cart, checkout).
2. Review relevant Toolshop documentation in https://github.com/testsmith-io/practice-software-testing/tree/main/docs and identify expected behavior for that flow.
3. Define test objectives, identify potential failure points, and choose suitable test activities for this flow.
4. Execute a focused mini-test session and document expected vs. actual outcomes.
5. Summarize results and propose one concrete quality action for the sprint.

## 6. Expected Outcome / Deliverable
One markdown artifact containing test objectives, selected test activities, findings, and one sprint action.

## 7. Timebox
30 minutes

## 8. Hints (Optional)
- Keep the scope to one user journey.
- Record assumptions explicitly if documentation is incomplete.
- Focus on evidence that helps team decisions.
