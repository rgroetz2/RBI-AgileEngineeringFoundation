# Apply Testing Throughout the Software Development Lifecycle on Toolshop

## 1. Title
Apply Testing Throughout the Software Development Lifecycle on Toolshop

## 2. Syllabus Reference
ISTQB FL – 2 Testing Throughout the Software Development Lifecycle

## 3. Learning Objective
Apply lifecycle-oriented testing decisions to one Toolshop feature so testing is planned and executed at the right stages.

## 4. Context / Scenario
In the current Toolshop sprint, the team needs better alignment between development activities and testing activities. You support this by mapping testing across the SDLC for one feature.

## 5. Task Instructions
1. Choose one Toolshop feature flow at https://practicesoftwaretesting.com/ (for example: login, cart, or checkout).
2. Use Toolshop docs and sprint context to identify where this feature sits in the lifecycle.
3. Define what testing should happen at different lifecycle points (before coding, during coding, after integration, before release).
4. Add one shift-left and one feedback improvement action for the team.
5. Capture risks if the proposed lifecycle testing is not followed.

## 6. Expected Outcome / Deliverable
A lifecycle-based test activity map for one Toolshop feature, including timing, responsibilities, and risks.

## 7. Timebox
30 minutes

## 8. Hints (Optional)
- Keep your mapping practical for one sprint.
- Use simple role labels (tester, developer, PO).
- Focus on actionable lifecycle decisions.
