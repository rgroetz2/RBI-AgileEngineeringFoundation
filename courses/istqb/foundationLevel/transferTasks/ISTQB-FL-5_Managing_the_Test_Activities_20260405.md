# Apply Managing the Test Activities on Toolshop

## 1. Title
Apply Managing the Test Activities on Toolshop

## 2. Syllabus Reference
ISTQB FL – 5 Managing the Test Activities

## 3. Learning Objective
Apply test management practices to one Toolshop sprint scope and produce a practical plan for control, reporting, and risk handling.

## 4. Context / Scenario
The Toolshop Scrum team needs a lightweight but effective way to plan and steer testing during the sprint.

## 5. Task Instructions
1. Select one Toolshop feature or flow planned for this sprint.
2. Define a mini test plan: scope, approach, priorities, entry/exit criteria, and key risks.
3. Propose 3 tracking metrics and a simple status communication format for stakeholders.
4. Add one configuration-management and one defect-management rule for this scope.
5. Summarize how you would monitor and adapt testing if time pressure increases.

## 6. Expected Outcome / Deliverable
A concise sprint test management sheet covering planning, risk, monitoring, and reporting decisions.

## 7. Timebox
30 minutes

## 8. Hints (Optional)
- Keep it realistic for one sprint team.
- Prioritize transparency and decision support.
- Tie each metric to an action.
