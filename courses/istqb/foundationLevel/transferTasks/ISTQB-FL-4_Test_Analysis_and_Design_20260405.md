# Apply Test Analysis and Design on Toolshop

## 1. Title
Apply Test Analysis and Design on Toolshop

## 2. Syllabus Reference
ISTQB FL – 4 Test Analysis and Design

## 3. Learning Objective
Create a focused, high-value test design for one Toolshop feature using appropriate ISTQB test techniques.

## 4. Context / Scenario
Your team needs a compact but effective test set for a Toolshop feature scheduled in the current sprint.

## 5. Task Instructions
1. Select one Toolshop feature flow at https://practicesoftwaretesting.com/.
2. Identify relevant requirements from the Toolshop documentation.
3. Choose suitable test techniques (for example: black-box, white-box-informed assumptions, or experience-based) and design a minimal test set.
4. Execute at least 2 designed tests and document observed outcomes.
5. Update the test set based on findings and residual risk.

## 6. Expected Outcome / Deliverable
A markdown test design package containing selected techniques, test cases, execution notes, and risk-based prioritization.

## 7. Timebox
30 minutes

## 8. Hints (Optional)
- Keep your design traceable to requirements.
- Prefer fewer, high-value tests over long lists.
- Note assumptions where implementation details are unknown.
