# Apply Static Testing on Toolshop

## 1. Title
Apply Static Testing on Toolshop

## 2. Syllabus Reference
ISTQB FL – 3 Static Testing

## 3. Learning Objective
Use static testing on Toolshop work products to detect defects early and reduce downstream rework.

## 4. Context / Scenario
Before executing dynamic tests, your Scrum team wants an early quality check of a Toolshop-related work product.

## 5. Task Instructions
1. Select one Toolshop work product to review (for example: requirement text, user story, acceptance criteria, or test case draft).
2. Open supporting docs from https://github.com/testsmith-io/practice-software-testing/tree/main/docs.
3. Perform a structured static review and capture defects, ambiguities, and missing information.
4. Classify findings by severity and likely impact on testing/development.
5. Propose one improvement to make future reviews more effective.

## 6. Expected Outcome / Deliverable
A concise review log with findings, severity, and follow-up actions.

## 7. Timebox
30 minutes

## 8. Hints (Optional)
- Focus on clarity, consistency, and testability.
- Do not execute the application for this task.
- Keep findings specific and actionable.
