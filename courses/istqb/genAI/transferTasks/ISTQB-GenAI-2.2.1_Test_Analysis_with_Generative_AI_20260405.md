### 1. Title
Generate Risk-Based Test Conditions from Feature Information

---

### 2. Syllabus Reference
ISTQB GenAI – 2.2.1 Test Analysis with Generative AI

---

### 3. Learning Objective
Use GenAI to derive and refine test conditions from feature descriptions and risk context.

---

### 4. Context / Scenario
You need fast test analysis for a sprint feature and want AI support without losing traceability.

---

### 5. Task Instructions
1. Select one Toolshop feature description from project docs.
2. Prompt GenAI to generate test conditions with risk and priority labels.
3. Review and remove non-verifiable or duplicate conditions.
4. Link each final condition to source requirement text.
5. Mark top 5 conditions for immediate test design.

---

### 6. Expected Outcome / Deliverable
Traceable, prioritized test-condition list.

---

### 7. Timebox
25 minutes

---

### 8. Hints (Optional)
- Ask AI to state assumptions explicitly.
- Validate conditions against real system behavior when possible.
