### 1. Title
Create a Threat Model for AI-Based Testing Tools

---

### 2. Syllabus Reference
ISTQB GenAI – 3.2.2 Vulnerabilities in AI-based Testing Tools

---

### 3. Learning Objective
Identify common vulnerability patterns and define controls for AI-enabled QA tooling.

---

### 4. Context / Scenario
Your organization plans broader AI tooling adoption and needs a targeted security assessment for test environments.

---

### 5. Task Instructions
1. Identify at least 5 threat scenarios (e.g., prompt injection, plugin abuse, credential leakage, poisoned context).
2. For each scenario, assess likelihood and impact.
3. Define preventive controls and detection signals.
4. Assign responsible role for each control.
5. Prioritize top 3 immediate actions.

---

### 6. Expected Outcome / Deliverable
Threat model table with prioritized mitigation actions.

---

### 7. Timebox
25 minutes

---

### 8. Hints (Optional)
- Include both technical and operational attack surfaces.
- Keep each control measurable.
