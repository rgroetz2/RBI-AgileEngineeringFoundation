### 1. Title
Build a Prompt-Technique Decision Guide for QA Tasks

---

### 2. Syllabus Reference
ISTQB GenAI – 2.2.5 Selecting Prompting Techniques for Software Testing

---

### 3. Learning Objective
Create practical selection criteria for choosing prompting techniques per test task type.

---

### 4. Context / Scenario
Testers use mixed prompting approaches with inconsistent outcomes. The team needs a decision aid.

---

### 5. Task Instructions
1. Define selection criteria (complexity, precision needed, context volume, explainability).
2. Map at least 5 test tasks to recommended techniques.
3. Add one anti-pattern per task.
4. Define fallback strategy if output quality is insufficient.

---

### 6. Expected Outcome / Deliverable
A one-page decision guide for prompting strategy.

---

### 7. Timebox
20 minutes

---

### 8. Hints (Optional)
- Keep rules simple and immediately usable.
- Include examples with concrete task wording.
