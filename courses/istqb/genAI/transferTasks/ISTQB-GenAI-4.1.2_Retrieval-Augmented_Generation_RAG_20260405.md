### 1. Title
Define a Mini-RAG Pattern for Test Knowledge Retrieval

---

### 2. Syllabus Reference
ISTQB GenAI – 4.1.2 Retrieval-Augmented Generation (RAG)

---

### 3. Learning Objective
Design a retrieval-driven generation approach to improve factual grounding of AI test outputs.

---

### 4. Context / Scenario
The team wants AI answers grounded in project artifacts (requirements, test plans, defect history).

---

### 5. Task Instructions
1. Define source corpus (which documents are in scope).
2. Propose chunking and retrieval strategy.
3. Define answer constraints (citation required, “unknown if not found”).
4. Add quality criteria for retrieval relevance and faithfulness.
5. Create one example prompt using the proposed RAG pattern.

---

### 6. Expected Outcome / Deliverable
RAG concept note with one worked example.

---

### 7. Timebox
30 minutes

---

### 8. Hints (Optional)
- Treat retrieved context as the primary source of truth.
- Require source references in outputs.
