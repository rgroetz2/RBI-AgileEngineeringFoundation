### 1. Title
Create an Environment Verification Plan for Toolshop Automation

---

### 2. Syllabus Reference
ISTQB TAE – 7.1.1 Plan to Verify the Test Automation Environment Including Test Tool Setup

---

### 3. Learning Objective
Plan verification activities that confirm the automation environment is ready, stable, and correctly configured.

---

### 4. Context / Scenario
The team often starts test execution before confirming environment readiness, resulting in avoidable failures.

---

### 5. Task Instructions
1. Define setup verification categories: tooling, environment access, test data, secrets, reporting artifacts.
2. Create at least 10 verification checks with expected result and failure action.
3. Mark each check as pre-run, in-run, or post-run.
4. Add one “stop execution” criterion and one “continue with warning” criterion.
5. Define who performs verification and where evidence is stored.

---

### 6. Expected Outcome / Deliverable
A structured environment verification plan/checklist with criteria and ownership.

---

### 7. Timebox
30 minutes

---

### 8. Hints (Optional)
Include a health check for SUT availability before executing test suites.
