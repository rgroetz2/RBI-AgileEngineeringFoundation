### 1. Title
Perform a Lightweight Tool Selection for Toolshop Automation

---

### 2. Syllabus Reference
ISTQB TAE – 1.2.2 Select Suitable Test Automation Tools For a Given System Under Test

---

### 3. Learning Objective
Apply a structured tool selection approach based on Toolshop technical and team constraints. Produce a transparent decision with criteria and trade-offs.

---

### 4. Context / Scenario
Your team wants one UI automation tool and one API automation tool for Toolshop regression in upcoming sprints.

---

### 5. Task Instructions
1. Explore Toolshop workflows in the browser and identify key test needs (UI, API, data, reporting, CI execution).
2. Create a criteria list: language fit, CI integration, maintainability, learning curve, debugging support, cost/license.
3. Compare 3 UI tool options and 2 API tool options (you may use known tools from your organization).
4. Score each option from 1 to 5 for each criterion.
5. Select one preferred stack and document two risks plus mitigations.

---

### 6. Expected Outcome / Deliverable
A scored comparison table and a final recommendation for UI and API tooling for Toolshop.

---

### 7. Timebox
30 minutes

---

### 8. Hints (Optional)
Do not optimize only for feature count; prioritize team skill alignment and pipeline compatibility.
