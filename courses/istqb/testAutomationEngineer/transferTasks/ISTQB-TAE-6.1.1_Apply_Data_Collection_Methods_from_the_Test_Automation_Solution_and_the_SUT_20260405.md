### 1. Title
Define Automation and SUT Data Collection for Toolshop

---

### 2. Syllabus Reference
ISTQB TAE – 6.1.1 Apply Data Collection Methods from the Test Automation Solution and the System Under Test

---

### 3. Learning Objective
Specify what execution and SUT data should be collected to support meaningful test analysis.

---

### 4. Context / Scenario
Your team has test pass/fail counts but cannot explain root causes or trends.

---

### 5. Task Instructions
1. Define data points from automation runs (duration, pass/fail, flaky reruns, failure type, environment).
2. Define data points from SUT behavior (response times, error states, UI failure signatures).
3. Create a collection method per data point (log parsing, report export, manual capture).
4. Mark frequency (per run, daily, sprint).
5. Add one data quality rule for completeness and one for consistency.

---

### 6. Expected Outcome / Deliverable
A data collection plan that combines automation and Toolshop SUT evidence.

---

### 7. Timebox
30 minutes

---

### 8. Hints (Optional)
Collect only data that supports a concrete decision or improvement action.
