### 1. Title
Specify Correct Behavior for a Toolshop Automated Test Suite

---

### 2. Syllabus Reference
ISTQB TAE – 7.1.2 Explain the Correct Behavior for a Given Automated Test Script and/or Test Suite

---

### 3. Learning Objective
Define clear expected behavior of automated scripts, including setup, assertions, and failure handling.

---

### 4. Context / Scenario
A new team member can run scripts but cannot explain expected suite behavior when failures occur.

---

### 5. Task Instructions
1. Select one Toolshop suite scope: login + add-to-cart + cart total validation.
2. Document expected behavior for each phase: preconditions, execution steps, assertions, teardown.
3. Define expected logging and artifact behavior on pass and on fail.
4. Add expected retry behavior (if any) and rule when retry is not allowed.
5. Write 3 acceptance criteria that determine whether the suite behavior is correct.

---

### 6. Expected Outcome / Deliverable
A behavior specification sheet for one automated suite, including pass/fail expectations.

---

### 7. Timebox
30 minutes

---

### 8. Hints (Optional)
Correct behavior includes deterministic outcomes and consistent artifact generation.
