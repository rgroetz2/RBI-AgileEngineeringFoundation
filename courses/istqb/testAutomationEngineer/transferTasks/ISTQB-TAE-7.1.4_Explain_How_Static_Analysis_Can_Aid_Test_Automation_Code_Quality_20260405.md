### 1. Title
Use Static Analysis Criteria to Improve Toolshop Test Code Quality

---

### 2. Syllabus Reference
ISTQB TAE – 7.1.4 Explain How Static Analysis Can Aid Test Automation Code Quality

---

### 3. Learning Objective
Apply static analysis thinking to detect quality issues early in test automation code.

---

### 4. Context / Scenario
The team spends too much time fixing avoidable script issues discovered only during execution.

---

### 5. Task Instructions
1. Define a static review checklist for automation code with at least 10 checks.
2. Include categories: naming/readability, duplication, dead code, hard waits, assertion quality, error handling.
3. Apply the checklist to one pseudo-code or existing test snippet used for Toolshop flows.
4. Record all detected issues and assign severity (Minor/Major/Critical).
5. Select top 3 issues and write concrete remediation actions.

---

### 6. Expected Outcome / Deliverable
A static-analysis review report with detected issues and prioritized fixes.

---

### 7. Timebox
30 minutes

---

### 8. Hints (Optional)
Flag hard-coded waits and duplicated selectors as high-value improvement targets.
