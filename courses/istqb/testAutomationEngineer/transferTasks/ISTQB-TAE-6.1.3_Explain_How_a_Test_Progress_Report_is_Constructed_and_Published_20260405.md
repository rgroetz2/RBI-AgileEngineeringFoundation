### 1. Title
Construct and Publish a Sprint Test Progress Report for Toolshop

---

### 2. Syllabus Reference
ISTQB TAE – 6.1.3 Explain How a Test Progress Report is Constructed and Published

---

### 3. Learning Objective
Build a concise, stakeholder-ready progress report that combines status, trends, and risks.

---

### 4. Context / Scenario
The Product Owner needs a clear sprint-level quality status for Toolshop before release decisions.

---

### 5. Task Instructions
1. Define report audience: Product Owner, Scrum Master, Developers, Testers.
2. Create report sections: Scope, Execution Summary, Defect Summary, Risks/Blockers, Trend, Next Actions.
3. Populate each section with sample values from your current or hypothetical sprint data.
4. Add one visual indicator per section (e.g., RAG status or trend arrow).
5. Define publication cadence and channel (e.g., end-of-day in sprint channel + review meeting).

---

### 6. Expected Outcome / Deliverable
A one-page test progress report template filled with sprint-relevant sample content.

---

### 7. Timebox
30 minutes

---

### 8. Hints (Optional)
Keep the report decision-oriented: highlight implications, not only counts.
