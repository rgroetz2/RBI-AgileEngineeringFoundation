### 1. Title
Define Configuration Management Rules for Toolshop Testware

---

### 2. Syllabus Reference
ISTQB TAE – 5.1.2 Explain Configuration Management for Testware

---

### 3. Learning Objective
Establish clear versioning, branching, and traceability rules for reliable testware changes.

---

### 4. Context / Scenario
The team has test scripts, test data, and environment files, but no consistent configuration management policy.

---

### 5. Task Instructions
1. List testware items to control: scripts, test data, config files, pipeline definitions, reports.
2. Define naming and versioning conventions for each item type.
3. Propose a branch strategy for testware aligned with sprint delivery.
4. Define traceability links between user stories, test cases, and automated scripts.
5. Add one change control rule and one rollback rule.

---

### 6. Expected Outcome / Deliverable
A concise configuration management policy for Toolshop testware.

---

### 7. Timebox
30 minutes

---

### 8. Hints (Optional)
Treat test data and environment configs as versioned artifacts, not informal files.
