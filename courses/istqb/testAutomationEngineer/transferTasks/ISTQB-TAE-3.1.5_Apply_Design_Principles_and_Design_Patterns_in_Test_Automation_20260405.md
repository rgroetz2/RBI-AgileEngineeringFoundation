### 1. Title
Apply Design Principles and Patterns to a Toolshop Test Flow

---

### 2. Syllabus Reference
ISTQB TAE – 3.1.5 Apply Design Principles and Design Patterns in Test Automation

---

### 3. Learning Objective
Improve test design quality by applying clear principles and reusable patterns to an automation scenario.

---

### 4. Context / Scenario
Existing Toolshop tests are brittle due to duplicated selectors and mixed responsibilities.

---

### 5. Task Instructions
1. Pick one scenario: login and add a product to cart.
2. Sketch a “before” structure that mixes page actions, waits, and assertions.
3. Refactor conceptually using Page Object + Test Data Builder (or equivalent pattern).
4. Apply at least three principles: single responsibility, DRY, and readable intent.
5. List two maintainability gains and one residual risk.

---

### 6. Expected Outcome / Deliverable
A before/after pseudo-design with identified principles, applied patterns, and impact summary.

---

### 7. Timebox
30 minutes

---

### 8. Hints (Optional)
If two tests repeat the same UI steps, extract an action method rather than duplicating code.
