### 1. Title
Plan Environment-Specific Automation Usage for Toolshop

---

### 2. Syllabus Reference
ISTQB TAE – 2.1.2 Explain How Test Automation is Leveraged within Different Environments

---

### 3. Learning Objective
Differentiate how automation should be applied in local, integration, and staging environments for Toolshop.

---

### 4. Context / Scenario
The team currently runs all tests in one environment and receives unreliable feedback.

---

### 5. Task Instructions
1. Define three environments: Local/Dev, Integration, and Staging.
2. Pick 6 Toolshop test ideas (e.g., search, cart, login, checkout, profile update, contact form).
3. Assign each test idea to one or more environments with rationale.
4. For each environment, specify execution trigger (commit, nightly, pre-release) and data strategy.
5. Mark one test that should not run in each environment and explain why.

---

### 6. Expected Outcome / Deliverable
An environment matrix showing test scope, trigger, and data strategy per environment.

---

### 7. Timebox
30 minutes

---

### 8. Hints (Optional)
Fast, deterministic checks belong early; broader end-to-end checks belong later.
