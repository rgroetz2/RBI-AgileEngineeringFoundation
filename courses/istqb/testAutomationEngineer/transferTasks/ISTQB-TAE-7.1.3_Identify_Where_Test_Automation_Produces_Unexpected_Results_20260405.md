### 1. Title
Diagnose Unexpected Automation Results in Toolshop Tests

---

### 2. Syllabus Reference
ISTQB TAE – 7.1.3 Identify Where Test Automation Produces Unexpected Results

---

### 3. Learning Objective
Pinpoint where unexpected outcomes originate and separate SUT issues from automation issues.

---

### 4. Context / Scenario
Your latest Toolshop regression run failed in multiple places, but root causes are unclear.

---

### 5. Task Instructions
1. Select 3 unexpected-result examples (real or simulated): false fail, false pass, inconsistent timing.
2. For each example, perform a triage using evidence categories: logs, screenshots, test data, environment state.
3. Identify the most probable source: script logic, assertion quality, test data, SUT defect, or environment.
4. Assign confidence level (High/Medium/Low) to each diagnosis.
5. Define one next diagnostic action per example.

---

### 6. Expected Outcome / Deliverable
A triage table with diagnosis, confidence, and follow-up action for 3 unexpected results.

---

### 7. Timebox
30 minutes

---

### 8. Hints (Optional)
Look first for unstable waits and data assumptions when diagnosing false failures.
