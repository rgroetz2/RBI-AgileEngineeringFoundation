### 1. Title
Map Test Automation Activities Across Scrum Sprint Events

---

### 2. Syllabus Reference
ISTQB TAE – 1.2.1 Explain How Test Automation is Applied Across Different Software Development Lifecycle Models

---

### 3. Learning Objective
Translate SDLC automation theory into a practical Scrum-oriented activity map for Toolshop. Show when and where automation work happens during a sprint.

---

### 4. Context / Scenario
Toolshop is developed in Scrum. A new team member assumes automation only happens after coding is complete.

---

### 5. Task Instructions
1. Open the Toolshop app and identify 3 user journeys (e.g., browse category, add product to cart, update account data).
2. Use sprint context from https://github.com/testsmith-io/practice-software-testing/tree/main/sprint5.
3. Create a timeline with these Scrum events: Backlog Refinement, Sprint Planning, Daily Scrum, Development, Review, Retrospective.
4. For each event, add at least one automation activity (e.g., define acceptance checks, automate API checks, stabilize data, review flaky tests).
5. Add one anti-pattern per event (what to avoid).

---

### 6. Expected Outcome / Deliverable
A one-page Scrum automation activity map with event-based actions and anti-patterns.

---

### 7. Timebox
60 minutes

---

### 8. Hints (Optional)
Include automation design early (refinement/planning), not only execution late in the sprint.
