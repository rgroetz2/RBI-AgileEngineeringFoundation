### 1. Title
Analyze Toolshop Test Result Data for Actionable Insights

---

### 2. Syllabus Reference
ISTQB TAE – 6.1.2 Analyze Data from the Test Automation Solution and the System Under Test to Better Understand Test Results

---

### 3. Learning Objective
Interpret result patterns and distinguish product defects from test or environment problems.

---

### 4. Context / Scenario
Recent Toolshop runs show inconsistent failures and the team needs evidence-based interpretation.

---

### 5. Task Instructions
1. Collect or assume a mini data set of 10 test results with fields: test name, status, duration, failure message, rerun outcome.
2. Categorize each failure as Product Defect, Test Script Issue, Environment Issue, or Data Issue.
3. Identify top 2 recurring failure patterns.
4. Propose one corrective action per pattern.
5. Summarize what should be escalated to development and what should be fixed in testware.

---

### 6. Expected Outcome / Deliverable
A short analysis report with categorized failures, patterns, and prioritized actions.

---

### 7. Timebox
30 minutes

---

### 8. Hints (Optional)
Use rerun behavior carefully: a pass-on-rerun can indicate timing or environment issues.
