## 1. Purpose

This Sprint Test Plan is derived from the Product Test Plan and focuses on the ESG features delivered in the current sprint.

The goal is to validate the ESG features and verify that they function correctly, while ensuring a fast and pragmatic testing approach to provide early feedback.

---

## 2. Scope

### In Scope
- CO₂ Impact Rating Display  
- Sustainability Badge System  
- ESG Mission Statement Integration  
- Imperial Unit Display  
- ESG Code of Conduct Display  
- Functional testing of ESG features  
- Basic regression testing of affected areas  

### Out of Scope
- Performance testing  
- Security testing  
- API testing  
- Mobile testing  

---

## 3. Test Objects

The following ESG features are tested:

- CO₂ Impact Rating Display  
- Sustainability Badge System  
- ESG Mission Statement Integration  
- Imperial Unit Display  
- ESG Code of Conduct Display  

---

## 4. Test Approach

The testing approach is based on:
- Requirements-based testing  
- Use case-based testing  
- Risk-based prioritization  

Testing is performed on UI level.  
Test execution is carried out in Testiny.

---

## 5. Test Design (T1–T5)

Test cases are designed using the T1–T5 model:

- T1: Happy path  
- T2: Alternative scenarios  
- T3: Exception cases  
- T4: Negative testing  
- T5: Misuse scenarios  

For each ESG feature, at least one T1 test case is defined.  
For higher-risk features, additional T2–T5 test cases are created.

---

## 6. Entry Criteria

Testing starts when:
- User stories are available  
- Test cases are created  
- Test environment is ready  
- Test data is available  

---

## 7. Exit Criteria

Testing is complete when:
- All planned test cases are executed  
- Critical defects are documented  
- No blocking defects remain open  
- Test run is completed  
- A test report is available
