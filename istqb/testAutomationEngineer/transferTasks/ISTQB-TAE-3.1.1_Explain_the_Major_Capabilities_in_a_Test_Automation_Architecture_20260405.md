### 1. Title
Define Core Capabilities of a Toolshop Test Automation Architecture

---

### 2. Syllabus Reference
ISTQB TAE – 3.1.1 Explain the Major Capabilities in a Test Automation Architecture

---

### 3. Learning Objective
Identify essential architecture capabilities and connect them to practical Toolshop testing needs.

---

### 4. Context / Scenario
The team has scripts but no explicit architecture, causing duplication and inconsistent reporting.

---

### 5. Task Instructions
1. Define capability blocks: Test Case Design, Test Data Management, Execution Engine, Assertions, Reporting, Logging, Environment Configuration, CI Integration...
2. For each block, write one Toolshop-specific requirement.
3. Map one concrete example flow (login + add-to-cart) across all capability blocks.
4. Identify the two capability gaps with highest delivery risk.
5. Suggest one improvement per gap.

---

### 6. Expected Outcome / Deliverable
A capability map showing architecture blocks (according to the Generic Test Automation Architecture), Toolshop-specific needs, and prioritized gaps.

---

### 7. Timebox
60 minutes

---

### 8. Hints (Optional)
Treat reporting and diagnostics as first-class capabilities, not optional extras.
