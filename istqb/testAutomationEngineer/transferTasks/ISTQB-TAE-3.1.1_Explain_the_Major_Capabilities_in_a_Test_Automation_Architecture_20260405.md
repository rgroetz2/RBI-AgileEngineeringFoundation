### 1. Title
(Creta Test Automation Strategy) - Define Core Capabilities of a Toolshop Test Automation Architecture

---

### 2. Syllabus Reference
ISTQB TAE – 3.1.1 Explain the Major Capabilities in a Test Automation Architecture

---

### 3. Learning Objective
Identify essential architecture blocks and other important things and connect them to practical Toolshop testing activities.

---

### 4. Context / Scenario
The team wants to start with test automation and needs a test automation strategy that defines where/what/how/when/by whom?

---

### 5. Task Instructions
1. Define capability blocks: Test Case Design, Test Data Management, Execution Engine, Assertions, Reporting, Logging, Environment Configuration, CI Integration...
2. For each block, write one Toolshop-specific requirement, tool to support this block, how GenAI can support this block.
3. Map one concrete example flow for first time customer(registration + login + add-to-cart + cheackout) across all capability blocks.
4. Identify the two capability gaps with highest delivery risk.
5. Suggest one improvement per gap.
6. Create the test automation strategy based on test-automation-strategy-template.md

---

### 6. Expected Outcome / Deliverable
Test Automation strategy as md-file with all chapters from test-automation-strategy-template.md showing architecture blocks (according to the Generic Test Automation Architecture), Toolshop-specific needs, and other sections.

---

### 7. Timebox
60 minutes

---

### 8. Hints (Optional)
Treat reporting and diagnostics as first-class capabilities, not optional extras.
