### 1. Title
Assess Maintainability Factors in Toolshop Test Automation

---

### 2. Syllabus Reference
ISTQB TAE – 4.3.1 Explain Which Factors Support and Affect Test Automation Solution Maintainability

---

### 3. Learning Objective
Evaluate maintainability drivers and anti-patterns, then define improvements for long-term testware health.

---

### 4. Context / Scenario
Toolshop UI changes in sprint increments have started breaking automated tests frequently.

---

### 5. Task Instructions
1. Create two lists: Maintainability Enablers and Maintainability Degraders.
2. Add at least 6 factors to each list (e.g., selector strategy, duplication, naming, modularity, data handling, review practices).
3. Rate current maturity for each factor as Good, Moderate, or Weak.
4. Choose the 3 weakest factors and define one concrete improvement action for each.
5. Add a measurable indicator to track each improvement over the next sprint.

---

### 6. Expected Outcome / Deliverable
A maintainability assessment sheet with prioritized improvement actions and tracking indicators.

---

### 7. Timebox
30 minutes

---

### 8. Hints (Optional)
Improvements are stronger when they include both coding standards and team working agreements.
